# 向地图添加数据 — 概述

> 原文：[Add your data to the map](https://docs.mapbox.com/android/maps/guides/add-your-data/)

## 首要选择：画在地图上方 vs 画在地图内部

| 方式 | 渲染位置 | 典型场景 |
|------|----------|----------|
| **Markers / Annotations / View Annotations** | 原生 UI 层，固定在地理坐标 | 标记点、弹窗、自定义控件 |
| **Style Layers** | 地图渲染管线内，与道路建筑同一套 | 大数据、热力图、与底图层级关系 |

## 地图如何渲染

Mapbox 地图在设备上用 **OpenGL** 客户端渲染，不是单张静态图。SDK 按当前视野从服务器取瓦片，在内存中组合多层后绘制。  
View 用 `MapView`，Compose 用 `MapboxMap`。

因此：平移、旋转、倾斜仍保持清晰；视野变化会动态加载新数据。

## 四种方式对比

### Markers（仅 Compose，实验性 API）

- 内置大头针样式，无需图片资源
- 适合快速加少量点
- 100+ 时性能下降；**Aura 若未启用 Compose 则不可用**

→ [06-Markers标记](./06-Markers标记.md)

### Annotations

- 点（需自备图标）、圆、线、面
- View + Compose 均支持；支持点击、拖拽、点聚合
- 数百个时性能变差

→ [07-Annotations标注](./07-Annotations标注.md)

### View Annotations

- 任意 `View` / Composable 叠在地图上
- 适合复杂 UI（按钮、卡片、ETA 条）
- 大量（>250）且允许重叠时性能差；无内置聚合

→ [08-View-Annotations](./08-View-Annotations视图标注.md)

### Style Layers

- GeoJSON、vector tiles 等 + circle/line/fill/symbol 等
- 万级要素、数据驱动样式、与底图层级精确控制
- 上手成本高；交互需 `queryRenderedFeatures` 等

→ [09-Style-Layers](./09-Style-Layers样式图层.md)、[04-源与图层](./04-源与图层.md)

## 选型清单

考虑：

1. 数据复杂度（几个点 vs 上万要素）
2. 交互（点击、拖拽、聚合）
3. 性能
4. 开发成本与是否已有 Compose

**经验法则**：

- Compose + 少量默认图钉 → Markers
- 自定义图标 / 线面 / View 项目 → Annotations
- 完全自定义 Android UI → View Annotations
- 大数据可视化 → Style Layers
