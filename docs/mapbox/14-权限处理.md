# 权限处理

> 原文：[Permission handling](https://docs.mapbox.com/android/maps/guides/user-location/permissions/)

Android 6.0（API 23）起，定位权限在**运行时**申请，不能只在 Manifest 声明。

## 推荐流程

1. 检查是否已授权
2. 未授权则请求
3. 根据结果启用或关闭定位功能

## 使用 PermissionsManager

```kotlin
lateinit var permissionsManager: PermissionsManager

override fun onCreate(savedInstanceState: Bundle?) {
    if (PermissionsManager.areLocationPermissionsGranted(this)) {
        // 已授权，启用定位
    } else {
        permissionsManager = PermissionsManager(object : PermissionsListener {
            override fun onExplanationNeeded(permissionsToExplain: List<String>) {
                // 可向用户说明为何需要定位
            }
            override fun onPermissionResult(granted: Boolean) {
                if (granted) {
                    // 启用定位
                } else {
                    // 降级或提示
                }
            }
        })
        permissionsManager.requestLocationPermissions(this)
    }
}

override fun onRequestPermissionsResult(
    requestCode: Int,
    permissions: Array<String>,
    grantResults: IntArray
) {
    permissionsManager.onRequestPermissionsResult(requestCode, permissions, grantResults)
}
```

**必须**把 `onRequestPermissionsResult` 转交给 `PermissionsManager`。

## 最佳实践

- 在**真正需要**定位时再申请，不要一启动就弹窗
- 拒绝后优雅降级，并可在合适时机用 `onExplanationNeeded` 解释原因
- `PermissionsManager` 也可用于其他 Android 权限
