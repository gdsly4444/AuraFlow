# 访问设备定位

> 原文：[Accessing device location](https://docs.mapbox.com/android/maps/guides/user-location/device-location/)

若应用需要位置信息但不一定要显示在地图上，可使用 **`LocationService`**，而不只依赖地图上的 location puck。

> 若只需 puck 在地图上的坐标，见 [Location provider](https://docs.mapbox.com/android/maps/api/)（`LocationComponentPlugin`）。

## LocationService

支持的 Provider：

- Google **Fused Location**
- Android **GPS / Network**

```kotlin
val locationService = LocationServiceFactory.getOrCreate()
val request = LocationProviderRequest.Builder()
    .interval(IntervalSettings.Builder().interval(0L).minimumInterval(0L).maximumInterval(0L).build())
    .displacement(0F)
    .accuracy(AccuracyLevel.HIGHEST)
    .build()

val result = locationService.getDeviceLocationProvider(request)
val locationProvider = if (result.isValue) result.value!! else null
```

## 订阅位置更新

```kotlin
val observer = object : LocationObserver {
    override fun onLocationUpdateReceived(locations: MutableList<Location>) {
        // 处理 locations
    }
}
locationProvider?.addLocationObserver(observer)

// 停止
locationProvider?.removeLocationObserver(observer)
```

也可用 `PendingIntent`：

```kotlin
locationProvider.requestLocationUpdates(myPendingIntent)
locationProvider.removeLocationUpdates(myPendingIntent)
```

带 `Looper` 的 `addLocationObserver(observer, looper)` 可指定回调线程。

**订阅机制**：第一个 observer 注册时开始采集，最后一个注销时停止。

## 最后一次已知位置

```kotlin
val cancelable = locationProvider.getLastLocation { result ->
    result?.let { /* 使用 */ }
}
cancelable.cancel()
```

## 与地图 puck 的区别

| | LocationService | LocationComponent（puck） |
|--|-----------------|---------------------------|
| 用途 | 业务逻辑、上报、导航算法 | 地图 UI 显示 |
| 数据 | 原始/融合定位 | 渲染用 + 可选 `LocationConsumer` |

地图上显示见 [16-地图上显示位置](./16-地图上显示位置.md)。
