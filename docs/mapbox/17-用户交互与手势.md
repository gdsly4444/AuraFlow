# 用户交互与手势

> 原文：[User Interaction](https://docs.mapbox.com/android/maps/guides/gestures/)

SDK 提供两类交互能力：

1. **默认手势**（平移、缩放、旋转等）— 基于 Mapbox Gestures for Android
2. **Interactions API** — 对样式中的 **featureset**（如建筑、地名）或地图本身绑定 `click` / `longClick`

---

## 默认手势

用户可通过手势操作地图：

- 平移（pan）
- 双指缩放（pinch zoom）
- 旋转（rotate）
- 倾斜（pitch）等

在 `MapView` 上通过 **Gestures Plugin** 可：

- 启用 / 禁用特定手势
- 调整手势行为（如缩放是否以焦点为中心）
- 监听手势事件（`OnMoveListener`、`OnScaleListener` 等）

典型用法（概念）：

```kotlin
mapView.gestures.apply {
    // 示例：禁用旋转
    rotateEnabled = false
    addOnMapClickListener { point ->
        // 点击地图经纬度
        true
    }
}
```

跟随用户位置时，常设置：

```kotlin
mapView.gestures.focalPoint = mapView.mapboxMap.pixelForCoordinate(userPoint)
```

使缩放围绕用户位置进行。

> 各手势开关与监听器名称以当前 SDK 的 `GesturesPlugin` API 文档为准。

---

## Interactions API（要素交互）

Mapbox Standard 等样式将可交互要素组织为 **featureset**（如 Buildings、Place Labels）。

可对 featureset 注册：

- `click`
- `longClick`

回调在用户点击对应要素时触发。也可对**地图空白处**注册交互，用于清除 feature state、重置高亮等。

**适用场景**：

- 点击建筑显示详情
- 点击 POI 标签
- 统一处理选中态与样式重置

> Interactions API 的完整代码示例见官方 [Examples](https://github.com/mapbox/mapbox-maps-android) 与 API Reference 中的 `Interactions` 相关类。

---

## 与标注 / 查询配合

| 需求 | 建议 |
|------|------|
| 点击 Annotation | `PointAnnotationManager.addClickListener` |
| 点击地图任意处 | `addOnMapClickListener` |
| 点击样式图层要素 | Interactions API 或 `queryRenderedFeatures` |
| Style Layer 拾取 | `mapboxMap.queryRenderedFeatures(screenPoint, options)` |
